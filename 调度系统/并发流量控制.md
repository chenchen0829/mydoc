## 维度

```
1. 应用并发数
2. 任务类型并发数(hive,shell,spark ...)
3. 调度类型并发数(正常调度，手动重刷，一次性任务，)
4. kpi任务并发数
5. 单个任务并发数(job维度)
6. 用户并发数
7. 时间范围
```



## 并发策略

```
1. 支持正向和反向
2. 支持树形，子策略
3. 支持单维度和多维度组合(多维度按照)， 单维度支持正反向。组合维度匹配按照(jobDim -> userDim, appDim, jobTypeDim , scheduleTypeDim, bizTypeDim)短路匹配。

```



## 设置

```
1. 可以在不同的时间段设置任务的并发度。(比如重跑任务的并发度在不同时间是不一样的。)
2. 在凌晨处理kpi任务时候，可以调整非kpi任务的执行并发度。(最大占用一半或者更少)
3. 限制单人重刷任务的并发度。
```
